# 🎓 College Student Spending Patterns — App Architecture

## What It Is
A web app where college students submit anonymous spending surveys, get personalized email reports, and explore a live public dashboard showing how their peers spend money. The entire data pipeline runs automatically.

---

## Tech Stack Overview

| Layer | Tool | Purpose |
|---|---|---|
| Frontend | Next.js + Shadcn | Dashboard UI |
| Auth | Clerk | Student accounts |
| Database | Supabase (PostgreSQL) | Store responses + analytics |
| Data Analysis | Python + Pandas + Matplotlib | Crunch the numbers |
| Email | Resend | Personalized reports |
| CI/CD | GitHub Actions | Auto-deploy + nightly pipeline |
| Hosting | Vercel | Live deployment |
| Domain | Namecheap | Custom URL |
| Design | Figma + Stitch | UI mockups |

## Database Schema (Supabase)

### `survey_responses`
```sql
id              UUID PRIMARY KEY
created_at      TIMESTAMP
year            TEXT        -- freshman, sophomore, etc.
major           TEXT
greek_life      BOOLEAN
has_job         BOOLEAN
monthly_income  INTEGER

-- Spending categories (monthly, in dollars)
rent            INTEGER
groceries       INTEGER
dining_out      INTEGER
transportation  INTEGER
subscriptions   INTEGER
entertainment   INTEGER
clothing        INTEGER
personal_care   INTEGER
misc            INTEGER
```

### `analytics_cache`
```sql
id              UUID PRIMARY KEY
updated_at      TIMESTAMP
category        TEXT        -- e.g. "dining_out"
cohort          TEXT        -- e.g. "sophomore", "greek_life", "all"
avg_spend       FLOAT
median_spend    FLOAT
percentile_25   FLOAT
percentile_75   FLOAT
sample_size     INTEGER
```

### `users` (via Clerk webhook)
```sql
clerk_id        TEXT PRIMARY KEY
email           TEXT
response_id     UUID REFERENCES survey_responses(id)
subscribed      BOOLEAN     -- weekly digest emails
```

## User Flows

### Flow 1 — Anonymous Survey Submission
```
1. Student visits site
2. Fills out 2-minute survey (no account required)
3. Optional: enters email to get personalized report
4. Response saved to Supabase
5. Resend triggers instant personalized email:
   "You spend $180/mo on dining — 34% above the Clemson average"
```

### Flow 2 — Authenticated Personal Dashboard
```
1. Student signs up with Clerk
2. Links their survey response to account
3. Sees personal dashboard:
   - Their spending vs. peer averages
   - Category breakdowns
   - Trend over time (if they resubmit each semester)
4. Opt into weekly digest emails via Resend
```

### Flow 3 — Public Dashboard
```
1. Anyone visits /dashboard
2. Sees live aggregate stats:
   - Average spend by category
   - Breakdowns by year, major, greek life
   - Charts generated by Matplotlib, served as images or
     rebuilt as interactive Recharts components
3. Updates nightly via GitHub Actions pipeline
```

## GitHub Actions Pipeline

### On Push to `main`
```yaml
- Install dependencies
- Run tests
- Deploy to Vercel
```

### Nightly Cron (12:00 AM)
```yaml
- Pull raw responses from Supabase
- Run Python/Pandas analysis script
- Generate updated aggregate stats
- Push stats back to analytics_cache table
- Generate Matplotlib charts → upload to Supabase Storage
- Send weekly digest emails via Resend (Sundays only)
```

## Python Analysis Pipeline

```
data/
├── pipeline.py          # Main script (run by GitHub Actions)
├── clean.py             # Pandas data cleaning
├── analyze.py           # Aggregate stats, percentiles, cohort splits
├── visualize.py         # Matplotlib chart generation
└── email_builder.py     # Build personalized email content
```

### Key Analyses
- Average spend per category (overall + by cohort)
- Percentile rankings (so students know where they stand)
- Correlation: income vs. spending, GPA proxy vs. spending
- Subscription audit: how much goes to Netflix, Spotify, etc.
- Outlier detection: flag unusual responses before including in averages
